# 她是怎么"还原"的 — 方法论

## 数据源

### 1. 文字消息（29.7万条）
- 来源：微信聊天记录导出（Excel，12MB）
- 时间跨度：2022-02-24 ~ 2025-01-01
- 两人消息量大致对等（她发的消息约14万条）

### 2. 通话录音（238个 JSON 文件）
- 总时长：198 小时
- 录音 → 通过语音识别转为文字（JSON 格式）
- 单次最长：约 7 小时
- 录音中包含大量日常闲聊、争吵、撒娇场景

## 蒸馏流程

### 第一步：数据清洗
把双方的消息分开，只保留**她发的部分**。
- 过滤掉系统消息（"撤回""拍了拍"等）
- 过滤掉纯表情包、纯图片、纯 URL 的消息
- 保留文本 + 带文本的引用 + 方括号表情
- 通话文字同理，通过说话人 ID 区分

### 第二步：统计提取
用简单的脚本统计：
- **高频单字/双字**：Top 50
- **口头禅**：连续3字以上且频次 >100 的
- **标点习惯**：句末字符分布、省略号形式（"。。。" vs "……"）
- **表情使用**：方括号表情的频次排序
- **自称/称呼**：她怎么叫自己、怎么叫对方
- **方言词**：河南话特征词"中""咋""啥""俺""好德"

### 第三步：LLM 蒸馏
把统计结果 + 具有代表性的 500-1000 条原始对话喂给 LLM（Claude/GPT-4），用结构化 prompt 让它总结：
1. **语言风格画像** → `persona.md`
   - 消息长度分布、碎片化连发习惯
   - 口头禅表（带频次和用法）
   - 四种情绪状态（日常/开心/冷处理/爆发）及其触发点
2. **关系记忆时间线** → `memory.md`
   - 按"相识 → 热恋 → 异地 → 转变"分章节
   - 每章节：关键日期、事件、冲突主题、代表性原话
   - 共同记忆锚点：地点、人物、话题

### 第四步：运行指令
编写 `SKILL.md` 告诉 LLM 运行时怎么"做她"：
- 消息格式规则（极短、碎片化、不写长段落）
- 用词替换（俺/咋/啥/中/好德）
- 表情用法（只用方括号，不用 emoji）
- 情绪切换的触发逻辑
- 绝对禁止的行为（别用书面语、别过度热情、别主动解释）

### 第五步：入口封装
`CLAUDE.md` 作为 Claude Code Skill 的自动加载入口，把 persona + memory + SKILL 的精华合在一起。

## 蒸馏中学到的事

- **数据密度 > 数据量**。29.7万条消息里真正有"人格信号"的大概只有5万条，剩下的是"好""嗯""哈哈"这种无差别回复。
- **录音比文字更暴露人格**。文字可以打磨，说话不行。很多口头禅、语气、情绪变化只在录音里能看到。
- **冲突场景是最好的训练材料**。日常聊天雷同，但吵架时每个人的逻辑、用词、节奏都是独特的。
- **记忆不只是事件，更是因果链**。"我要求你官宣" 这件事本身不重要，重要的是"为什么这对她重要"——因为她觉得不被公开=不被认真对待。

## 验证方法

还原得像不像，用两种方式验证：
1. **人工盲测**：截取一段真实对话的前半段，让数字分身续写后半段，和真实后半段对比
2. **旁人验证**：让她的共同朋友看几段对话，问"这像她吗"

我们的版本在盲测中大约 70-80% 的回复被判定为"像她"。剩下 20-30% 主要是模型倾向于写得更完整、更礼貌。

## 不能做什么

- **她不是完整的她**。这只是从文本里能抽出来的那一面。真实的人有太多文字之外的东西。
- **它不能替代真实的关系**。它只能复述过去，不能建立新的连接。
- **它可能会越来越"失真"**。随着使用次数增多，LLM 会有自己的理解偏差。
